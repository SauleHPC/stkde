\begin{abstract}
  The exponential growth of available data, caused by the BigData era,
  increased the interest in interactive exploratory analysis as dataset
  can no longer be understood by crawling manually through it and
  looking at a few statistics. In a Geographical Information Systems
  (GIS), the dataset is often composed of events localized in space
  and time; and visualizing such a dataset involves building a map of
  where the events occurred.

  We focus in this paper on events that are localized among three
  dimensions (latitude, longitude, and time), and on computing the
  first step of the visualization pipeline, space-time kernel density
  estimation, which is most computationally expensive.  Starting from
  a gold standard implementation, we show how algorithm design and
  engineering, parallel decomposition, and acceleration can be applied
  to bring real-time computing to space-time kernel density
  estimation. We validate our techniques on real world datasets
  extracted from infectious disease, social media, and ornithology.
\end{abstract}

\subsection{Problem Definition}


Notations: domainsize $gx$, $gy$, $gt$. bandwidth: $sbw$,
$tbw$. Spatial resolution $sres$, temporal resolution $tres$. This
forms a grid of size $GX=\ceil{\frac{gx}{sres}}$,
$GY=\ceil{\frac{gy}{sres}}$, $GT=\ceil{\frac{gt}{tres}}$. And we
denote the bandwidth in voxel $SBW=\ceil{\frac{sbw}{sres}}$,
$TBW=\ceil{\frac{tbw}{tres}}$.

There are $N$ observations.

\subsection{Observation-based Algorithm}

The voxel-based algorithm suffers from a massive cost of computing
distances. The gold implementation reduces them by extremely
decomposition the domain, but that still incures a significant
overhead. These calculation are mostly unnecessary since we know
exactly where each observation will radiate density. The observation
based algorithm OB leverages that property.

The algorithm OB incurs mostly two costs. Initializing the memory which costs
$\Theta(GX * GY * GT)$ and computing the impact of each observation
$\Theta(N * SBW^2 * TBW)$. Note that it is possible that either of these
two term is significantly larger than the other. Therefore the
complexity of the algorithm is $\Theta(GX * GY * GT + N * SBW^2 * TBW)$.

\subsection{Exploiting Symmetries}

When the bandwidth is large computing the density contribution of each
observation each voxel in the bandwidth is the most expensive
operation. Each of these $N * (2 * SBW)^2 * 2 * TBW$ calculations costs
approximately 40 floating point operations. (The exact cost is
difficult to estimate since there are floating point additions,
multiplications, divisions, and square root operations.)

\subsection{Domain Replication}

The simplest way to parallelize this kind of computation is to split
the observation equally among the $P$ different computational
units. Though if two close-by observations are computed
simultaneously, the cylinder of density around it might collide and
there is a race condition.

The SYM-DR algorithm solves the data race issue by having
each of the $P$ computational unit aggregate the result on a local
copy of the domain. Once all the observations have been processed, the
$P$ copies need to be summed.

This algorithm has a memory requirement of $\Theta (P * GX * GY * GT)$
and a parallel amount of work of $\Theta (P * GX * GY * GT + N * SBW^2
* TBW)$. Though the increase in work enable the parallelism to be
pleasingly parallel.

\subsection{Domain Decomposition}

An other strategy to avoid the data race is by computing the intensity
of the voxels of different subdomains independently. The SYM-DD
algorithm splits the domain in AxBxC subdomains. Then it associates to
each subdomain the observations which associated cylinder intersect on
a voxel of the subdomain.
